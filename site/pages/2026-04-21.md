# Page 56 — Maximum Size Subarray Sum = K (Prefix Sum + First Index Map)

Yesterday we counted how many subarrays hit sum `k`. Today’s variation is common in interviews:

Given `nums` and `k`, return the **length of the longest subarray** whose sum equals `k`.

This is where many candidates almost solve it, then lose points by storing the wrong thing in the hashmap.

## Core shift from “count” to “longest”

Prefix sum still drives everything.
Let `pref[i]` be sum from index `0..i`.
For a subarray `j+1..i` to sum to `k`:

`pref[i] - pref[j] = k`  ->  `pref[j] = pref[i] - k`

To maximize length `i - j`, once you’re at index `i`, you want the **earliest index `j`** where that needed prefix appeared.

So unlike yesterday’s frequency map, today we store:
- `first_index[prefix_sum] = earliest position where it appeared`

## Interview-grade Python

```python
def max_subarray_len(nums, k):
    first = {0: -1}  # prefix sum 0 before array starts
    pref = 0
    best = 0

    for i, x in enumerate(nums):
        pref += x

        target = pref - k
        if target in first:
            best = max(best, i - first[target])

        # Keep earliest index only
        if pref not in first:
            first[pref] = i

    return best
```

## Why “earliest only” matters

If you overwrite `first[pref]` with later indices, you shrink potential lengths and lose correct answers.

Quick example:
`nums = [1, -1, 5, -2, 3], k = 3`

Running prefix: `1, 0, 5, 3, 6`

At `i=3`, `pref=3`, target is `0`. Earliest index of prefix `0` is `-1`, so length is `3 - (-1) = 4` (`[1, -1, 5, -2]`).
If you had overwritten prefix `0` to index `1`, you’d get only length `2`, which is wrong.

## What to say out loud in interviews

“I’ll use prefix sums and map each prefix to its first occurrence index. At each index, I check whether `prefix-k` has appeared; if yes, that gives a candidate subarray ending here. Keeping only the first occurrence maximizes length. Time `O(n)`, space `O(n)`.”

## Common mistakes

1. Using a frequency map instead of earliest-index map.
2. Overwriting prefix indices (breaks longest logic).
3. Forgetting seed `0: -1` (misses subarrays starting at index 0).
4. Mixing this up with the counting version from yesterday.

Pattern memory: **exact sum + longest length** usually means **prefix sum + first occurrence hash map**.
