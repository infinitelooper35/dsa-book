# Page 42 — Kth Largest in a Stream: Min-Heap That Stays Interview-Safe

Yesterday’s median stream problem used two heaps. Today’s sibling pattern is simpler and shows up everywhere in interviews:

“Design a class that continuously receives numbers and returns the kth largest element so far.”

If you re-sort the full list after every insert, you’ll drown in `O(n log n)` updates. The practical answer is a **min-heap of size `k`**.

## Interview case

Design `KthLargest` with:
- constructor: `KthLargest(k, nums)`
- method: `add(val)` returns current kth largest element

## Core idea

Keep only the top `k` largest elements seen so far in a min-heap:
- heap top (`heap[0]`) is the **smallest** among those top `k`
- therefore, heap top is exactly the **kth largest overall**

Update rule on new value `x`:
1. Push `x` into heap.
2. If heap size exceeds `k`, pop once.

That pop removes the smallest value, so heap keeps the best `k` candidates.

## Python template

```python
import heapq


class KthLargest:
    def __init__(self, k: int, nums: list[int]):
        self.k = k
        self.heap = nums[:]
        heapq.heapify(self.heap)

        while len(self.heap) > k:
            heapq.heappop(self.heap)

    def add(self, val: int) -> int:
        heapq.heappush(self.heap, val)
        if len(self.heap) > self.k:
            heapq.heappop(self.heap)
        return self.heap[0]
```

Complexity:
- Constructor: `O(n log k)` (after heapify + trims)
- `add`: `O(log k)`
- Space: `O(k)`

Interview tip: if asked to support “remove arbitrary value” too, call out that this exact design is optimized for append-only streams; deletions need extra bookkeeping.

## What to say out loud

“I don’t need a fully sorted stream. I only need the top `k` elements at all times. So I keep a min-heap of size `k`; the root is the kth largest by definition.”

That explanation is short, correct, and very interview-friendly.

## Common mistakes

1. Using a max-heap of all elements (`O(n)` space, slower updates).
2. Forgetting to trim heap back to `k` after each insert.
3. Returning last inserted value instead of heap root.
4. Confusing kth largest with kth smallest (for kth smallest, use max-heap size `k`).

## Pattern extension

Same bounded-heap trick appears in:
- top-k frequent elements
- k closest points
- running leaderboard cutoffs

Interview trigger: if the prompt says “continuously maintain top-k,” think **fixed-size heap**, not repeated sorting.
