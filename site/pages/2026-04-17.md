# Page 52 — Constrained Subsequence Sum: DP + Monotonic Deque

Yesterday’s shortest-subarray page used prefix sums + a monotonic deque to handle negatives. Today we’ll use a similar deque trick on top of DP for another interview favorite where brute force dies fast.

Problem (LeetCode 1425 style):
Given `nums` and integer `k`, return the maximum sum of a **non-empty subsequence** such that for any two consecutive chosen indices `i < j`, we have `j - i <= k`.

This is not a subarray (you can skip elements), but jumps are limited.

## Why brute force fails

A naive DP checks all previous `k` positions each time:

`dp[i] = nums[i] + max(0, dp[i-1], dp[i-2], ..., dp[i-k])`

That recurrence is correct, but scanning `k` candidates for each `i` costs `O(nk)`.

## Key optimization

You only need the **maximum dp value in the last k indices**. That is a sliding-window maximum problem, so maintain a deque of indices whose `dp` values are decreasing.

For each index `i`:
1. Remove deque front if it is out of range (`< i-k`).
2. Best previous value is `dp[dq[0]]` if deque is not empty.
3. Compute `dp[i] = nums[i] + max(0, best_prev)`.
4. Remove smaller `dp` values from deque back.
5. Push `i`.

Each index is pushed and popped at most once -> `O(n)`.

## Interview-grade Python

```python
from collections import deque

def constrained_subset_sum(nums, k):
    n = len(nums)
    dp = [0] * n
    dq = deque()  # indices, dp decreasing
    ans = -10**18

    for i, x in enumerate(nums):
        # 1) evict out-of-window indices
        while dq and dq[0] < i - k:
            dq.popleft()

        # 2) best transition
        best_prev = dp[dq[0]] if dq else 0
        dp[i] = x + max(0, best_prev)
        ans = max(ans, dp[i])

        # 3) maintain decreasing dp deque
        while dq and dp[dq[-1]] <= dp[i]:
            dq.pop()
        dq.append(i)

    return ans
```

## What to say out loud

“I’ll use DP where each state depends on the max DP value in the last `k` indices. To avoid `O(k)` scans, I maintain that max with a monotonic deque, reducing total complexity to linear.”

## Common mistakes

1. Treating this like Kadane’s (no distance constraint).
2. Forgetting `max(0, best_prev)`, which can force harmful extensions.
3. Returning `max(dp)` only at the end without tracking safely for all-negative arrays.
4. Eviction bug: use `< i-k`, not `<= i-k`.

Pattern memory: if DP transition says “max over recent window,” think **DP + monotonic deque** immediately.
