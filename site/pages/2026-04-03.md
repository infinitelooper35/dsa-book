# Page 38 — Find Median from Data Stream: Two Heaps You Can Explain in 30 Seconds

Yesterday’s sliding-window median used two heaps plus lazy deletion. Today we simplify to the base pattern interviewers ask first: **Median Finder / Data Stream Median**.

Prompt style:
- “Design a data structure with `addNum(x)` and `findMedian()`.”
- Numbers arrive one by one.
- Return median at any point.

If you sort on every insertion, you get `O(n log n)` per add (or `O(n)` insert with binary search + array shift). That’s too slow in interviews once they push for many operations.

## Core design

Use two heaps:
- `small`: max-heap for lower half (Python: push negatives)
- `large`: min-heap for upper half

Balance rule:
1. `len(small) == len(large)` OR
2. `len(small) == len(large) + 1`

This keeps median retrieval O(1):
- odd count → top of `small`
- even count → average of tops

## Interview-safe implementation

```python
import heapq

class MedianFinder:
    def __init__(self):
        self.small = []  # max-heap via negatives
        self.large = []  # min-heap

    def addNum(self, num: int) -> None:
        # Step 1: always push into small first
        heapq.heappush(self.small, -num)

        # Step 2: ordering fix: every small <= every large
        if self.large and -self.small[0] > self.large[0]:
            x = -heapq.heappop(self.small)
            heapq.heappush(self.large, x)

        # Step 3: size rebalance
        if len(self.small) > len(self.large) + 1:
            heapq.heappush(self.large, -heapq.heappop(self.small))
        elif len(self.large) > len(self.small):
            heapq.heappush(self.small, -heapq.heappop(self.large))

    def findMedian(self) -> float:
        if len(self.small) > len(self.large):
            return float(-self.small[0])
        return (-self.small[0] + self.large[0]) / 2.0
```

Complexity:
- `addNum`: `O(log n)`
- `findMedian`: `O(1)`
- Space: `O(n)`

## What to say out loud

“I keep lower half in a max-heap and upper half in a min-heap. I enforce ordering (`max(lower) <= min(upper)`) and size balance (difference at most 1). Median is either top of lower heap or average of both tops.”

That answer is crisp, implementable, and interview-friendly.

## Common mistakes

1. Rebalancing sizes but not enforcing cross-heap ordering.
2. Returning integer division on even count (`/` vs `//`).
3. Putting extra element in either heap inconsistently (pick one convention and stick to it).
4. Overcomplicating insertion logic; keep the 3-step routine deterministic.

If you can code this from memory in under 5 minutes, you’re in great shape for most median/order-statistics interview follow-ups.
