# Page 64 — Sliding Window Maximum with Monotonic Deque (The Queue Pattern That Actually Wins)

Yesterday’s monotonic **stack** solved “next greater to the right.” Today we switch to its interview sibling: monotonic **deque** for window queries.

Classic prompt:
“Given array `nums` and window size `k`, return the maximum in every window of length `k`.”

Example:
`nums = [1,3,-1,-3,5,3,6,7], k = 3` → `[3,3,5,5,6,7]`

## Why brute force fails

For each window, scan `k` elements to find max.
- Number of windows: `n - k + 1`
- Time: `O(nk)`

If `n = 1e5` and `k = 500`, you’re dead on runtime.

## Invariant that makes it O(n)

Maintain a deque of **indices** with two rules:
1. Indices are always inside current window.
2. Values at those indices are in **decreasing** order.

Meaning:
- Front of deque is always max for current window.
- Before pushing index `i`, pop from back while `nums[back] <= nums[i]` (those can never become max again).
- Pop front if it falls out of window (`<= i - k`).

Each index is pushed once, popped at most once → `O(n)`.

## Interview-grade Python

```python
from collections import deque

def max_sliding_window(nums, k):
    if k == 0 or not nums:
        return []

    dq = deque()  # indices, nums[dq] strictly decreasing
    ans = []

    for i, x in enumerate(nums):
        # Remove indices out of window
        while dq and dq[0] <= i - k:
            dq.popleft()

        # Maintain decreasing order
        while dq and nums[dq[-1]] <= x:
            dq.pop()

        dq.append(i)

        # Window is formed when i >= k - 1
        if i >= k - 1:
            ans.append(nums[dq[0]])

    return ans
```

Complexity:
- Time: `O(n)`
- Space: `O(k)` (deque size never exceeds window)

## What to say out loud

“I’ll keep candidate max indices in a decreasing deque. Front is current max. I remove expired indices from front and dominated values from back. This gives amortized constant work per element, total linear time.”

## High-value edge checks

- `k = 1` → output equals input
- `k = len(nums)` → single maximum
- duplicates like `[4,4,4]` with `k=2` (confirm `<=` pop rule)
- negative-only arrays

If you can explain *why popped elements are permanently useless*, interviewers know you understand the structure—not just memorized code.
