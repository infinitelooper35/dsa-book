# Page 50 — Longest Subarray Within Limit: Two Monotonic Deques, One Window

Yesterday you used one monotonic deque for sliding window maximum. Today’s interview upgrade is using **two deques at once** to track window validity.

Problem (LeetCode 1438 style):
Given `nums` and `limit`, return the length of the longest contiguous subarray where:

`max(nums[l..r]) - min(nums[l..r]) <= limit`

Brute force checks every subarray and recomputes max/min, which is too slow (`O(n^2)` or worse). Interview target is `O(n)`.

## Core idea

Maintain a sliding window `[l, r]` and keep:
- `maxdq`: indices with values in decreasing order (front = current max)
- `mindq`: indices with values in increasing order (front = current min)

At each `r`:
1. Push `r` into both deques while preserving monotonic order.
2. If window violates the rule (`nums[maxdq[0]] - nums[mindq[0]] > limit`), move `l` right until valid.
3. While moving `l`, pop fronts if those indices fall out of the window.
4. Update best length.

## Invariants to narrate in interview

- `maxdq[0]` always indexes the maximum in current window.
- `mindq[0]` always indexes the minimum in current window.
- Window is valid iff `nums[maxdq[0]] - nums[mindq[0]] <= limit`.

If you state these clearly, your while-loops feel intentional instead of “magic deque code.”

## Interview-grade Python

```python
from collections import deque

def longest_subarray(nums, limit):
    maxdq, mindq = deque(), deque()
    l = 0
    best = 0

    for r, x in enumerate(nums):
        while maxdq and nums[maxdq[-1]] <= x:
            maxdq.pop()
        maxdq.append(r)

        while mindq and nums[mindq[-1]] >= x:
            mindq.pop()
        mindq.append(r)

        while nums[maxdq[0]] - nums[mindq[0]] > limit:
            if maxdq[0] == l:
                maxdq.popleft()
            if mindq[0] == l:
                mindq.popleft()
            l += 1

        best = max(best, r - l + 1)

    return best
```

## Complexity (say this out loud)

- Time: `O(n)` (each index pushed/popped at most once per deque)
- Space: `O(n)` worst-case deque storage

## Pressure mistakes to avoid

1. Storing values instead of indices (you can’t evict by position cleanly).
2. Forgetting to shrink window in a `while` loop (single `if` is wrong for large violations).
3. Updating `best` before restoring validity.

This pattern is a top-tier signal: you can maintain dynamic constraints, not just run static scans.
