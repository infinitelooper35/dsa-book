# Page 39 — Quickselect: Find Kth Largest Without Sorting Everything

After two-heap median patterns, interviewers often pivot to a simpler but high-signal question:

“Find the kth largest element in an unsorted array.”

Most candidates sort immediately. That works, but it’s not always the best trade-off.

- Full sort: `O(n log n)`
- Heap (size `k`): `O(n log k)`
- **Quickselect (average): `O(n)`**

If the prompt asks for a single order statistic (not full ordering), Quickselect is usually the sharpest answer.

## Interview case

Given `nums` and integer `k`, return the kth largest element.

Example:
`nums = [3,2,1,5,6,4], k = 2` → `5`

## Core idea

Quickselect is partition-based (same backbone as Quicksort):
1. Pick a pivot.
2. Partition array into values `< pivot` and `>= pivot` (or the reverse, as long as consistent).
3. Check pivot’s final index.
   - If it matches target index, done.
   - Otherwise recurse/iterate only on the side that contains the target.

Unlike Quicksort, you throw away half the work each step, because you only care about one index.

For kth largest, convert to zero-based target:
`target = len(nums) - k` (kth largest == target-th smallest index).

## Python template (iterative, interview-safe)

```python
import random


def find_kth_largest(nums, k):
    target = len(nums) - k
    l, r = 0, len(nums) - 1

    while l <= r:
        pivot_idx = random.randint(l, r)
        pivot = nums[pivot_idx]

        # move pivot to end
        nums[pivot_idx], nums[r] = nums[r], nums[pivot_idx]

        # partition: < pivot on left
        p = l
        for i in range(l, r):
            if nums[i] < pivot:
                nums[p], nums[i] = nums[i], nums[p]
                p += 1

        # place pivot in final position
        nums[p], nums[r] = nums[r], nums[p]

        if p == target:
            return nums[p]
        elif p < target:
            l = p + 1
        else:
            r = p - 1
```

Complexity:
- Average: `O(n)`
- Worst case: `O(n^2)` (rare with randomized pivot)
- Extra space: `O(1)` iterative

## What to say out loud

“I’ll use Quickselect because we only need one rank, not full sorting. After each partition, I discard the half that cannot contain the target index.”

That line shows algorithmic judgment, not just coding speed.

## Common mistakes

1. Mixing up kth largest vs kth smallest index conversion.
2. Inconsistent partition rule (`<` vs `<=`) causing off-by-one bugs.
3. Recursing both sides (that becomes Quicksort by accident).
4. Forgetting this mutates the input array (mention if that matters).

Interview trigger: if prompt asks for one element by rank, think **Quickselect before full sort**.
