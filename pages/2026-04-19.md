# Page 54 — Shortest Subarray with Sum ≥ K (Prefix Sum + Monotonic Deque)

We’re still in monotonic-deque territory, but today’s problem is trickier than Jump Game VI because the array can contain **negative numbers**.

Problem (LeetCode 862 style):
Given `nums` and integer `k`, return the length of the shortest non-empty subarray with sum at least `k`. If none exists, return `-1`.

A normal sliding window fails here. With negatives, expanding right or shrinking left is no longer monotonic, so you can’t trust the usual two-pointer rules.

## Step 1: Reframe with prefix sums
Let `pref[i]` be sum of first `i` elements (`pref[0] = 0`).
Subarray sum from `j` to `i-1` is:
`pref[i] - pref[j]`

We need:
`pref[i] - pref[j] >= k`  ->  `pref[j] <= pref[i] - k`

For each `i`, we want the **earliest valid j** that also gives the **shortest length** `i - j`.

## Step 2: Maintain deque of candidate prefix indices
Keep indices in deque with **increasing prefix values**.
Why increasing? If `pref[a] >= pref[b]` and `a < b`, index `a` is dominated: it’s earlier but has a worse (larger) prefix, so it can never produce a shorter/better valid subarray later.

For each prefix index `i`:
1. While front is valid (`pref[i] - pref[dq[0]] >= k`), update answer and pop front (we try to shorten length).
2. While back has larger/equal prefix (`pref[dq[-1]] >= pref[i]`), pop back (dominated).
3. Push `i`.

Each index is pushed/popped at most once -> `O(n)`.

## Interview-grade Python

```python
from collections import deque

def shortest_subarray(nums, k):
    n = len(nums)
    pref = [0] * (n + 1)
    for i, x in enumerate(nums, 1):
        pref[i] = pref[i - 1] + x

    ans = n + 1
    dq = deque()  # indices of pref, pref values increasing

    for i in range(n + 1):
        while dq and pref[i] - pref[dq[0]] >= k:
            ans = min(ans, i - dq.popleft())

        while dq and pref[dq[-1]] >= pref[i]:
            dq.pop()

        dq.append(i)

    return ans if ans <= n else -1
```

## What to say out loud in interviews

“I converted subarray sums to prefix differences. For each `i`, I need the smallest `j` with `pref[j] <= pref[i] - k`, while minimizing `i - j`. A monotonic deque keeps only useful candidate prefix indices, giving linear time.”

## Common failure points

1. Using sliding window despite negatives.
2. Forgetting `pref[0] = 0`, which breaks subarrays starting at index 0.
3. Not popping dominated back elements (`>=` matters).
4. Returning 0 or `n+1` instead of `-1` when no valid subarray exists.

Pattern memory: when you see “shortest/longest subarray with condition” and negatives are allowed, think **prefix sums + monotonic deque**, not classic window.
