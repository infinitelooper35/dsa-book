# Page 43 — Merge K Sorted Lists: Heap Beats Pairwise Merging Under Pressure

Yesterday we used a fixed-size heap for a stream. Today we’ll use a heap again, but this time to merge already-sorted inputs.

Classic prompt:

“Given `k` sorted linked lists, merge them into one sorted linked list.”

A lot of candidates jump to repeated two-list merges. That works, but the cleaner interview answer is a **min-heap** holding one current node from each list.

## Interview framing

Each list is already sorted. So the smallest unmerged value across all lists must be at one of the current heads. If we can quickly grab the smallest head each step, we’re done.

That is exactly what a min-heap gives us.

## Heap strategy

1. Push `(node.val, list_index, node)` for each non-empty list head.
2. Pop smallest tuple.
3. Append that node to output.
4. If popped node has `next`, push `(next.val, list_index, next)`.
5. Repeat until heap is empty.

We include `list_index` as a tie-breaker so Python never tries to compare `ListNode` objects directly when values match.

## Python template

```python
import heapq


def merge_k_lists(lists):
    heap = []

    for i, node in enumerate(lists):
        if node:
            heapq.heappush(heap, (node.val, i, node))

    dummy = ListNode(0)
    tail = dummy

    while heap:
        _, i, node = heapq.heappop(heap)
        tail.next = node
        tail = tail.next

        if node.next:
            heapq.heappush(heap, (node.next.val, i, node.next))

    return dummy.next
```

Complexity:
- Let `N` = total nodes, `k` = number of lists
- Time: `O(N log k)`
- Space: `O(k)` heap

Why this wins interviews: you demonstrate you can exploit structure (`k` sorted sources) instead of brute-force flatten + sort (`O(N log N)`).

## What to say out loud

“I only need the smallest current head among `k` lists at any point. A min-heap of size at most `k` gives me that in `O(log k)` per extraction, so total `O(N log k)`.”

That one sentence is usually enough to justify the full design.

## Common mistakes

1. Flattening all nodes into an array and sorting again (misses optimization).
2. Forgetting tie-breaker in heap tuple (Python comparison error).
3. Creating new nodes unnecessarily (you can relink existing nodes).
4. Forgetting to advance/push `node.next` after pop.

If interviewer asks for arrays instead of linked lists, same pattern applies: this is the exact backbone of “merge k sorted arrays/streams.”
