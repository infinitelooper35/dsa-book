# Page 36 — Monotonic Queue: Sliding Window Maximum Without Re-Scanning

If monotonic stacks solved “next greater element,” monotonic queues solve a sibling problem interviewers love:

Given `nums` and window size `k`, return the maximum for every contiguous window.

Example:
`nums = [1,3,-1,-3,5,3,6,7], k = 3`
Output: `[3,3,5,5,6,7]`

## Why brute force loses

Brute force checks each window and scans `k` elements to find max.
- Windows: about `n`
- Work per window: `k`
- Total: `O(nk)`

That gets crushed when `n` is large and `k` is non-trivial.

## Core idea: decreasing deque of indices

Use a deque storing indices whose values are in decreasing order.
Front is always the max for the current window.

For each index `i`:
1. Remove indices from front if they are out of window (`<= i-k`).
2. Remove indices from back while `nums[back] <= nums[i]` (they can never be future max).
3. Push `i`.
4. If window is formed (`i >= k-1`), answer is `nums[deque[0]]`.

Each index enters and leaves deque at most once → `O(n)` time.

## Python template

```python
from collections import deque

def max_sliding_window(nums, k):
    if not nums or k == 0:
        return []

    dq = deque()  # indices, nums[dq] is decreasing
    ans = []

    for i, x in enumerate(nums):
        # 1) drop expired indices
        while dq and dq[0] <= i - k:
            dq.popleft()

        # 2) keep deque decreasing
        while dq and nums[dq[-1]] <= x:
            dq.pop()

        # 3) add current index
        dq.append(i)

        # 4) record max when first full window appears
        if i >= k - 1:
            ans.append(nums[dq[0]])

    return ans
```

Complexity:
- Time: `O(n)`
- Space: `O(k)` (deque never holds more than one window)

## Interview narration that scores points

“I’ll maintain a decreasing deque of candidate max indices. I evict expired indices from the front and evict dominated values from the back. The front is always the current window max.”

That line tells the interviewer you understand **candidate maintenance**, not just memorized code.

## High-frequency mistakes

1. Storing values instead of indices (can’t detect expiration correctly).
2. Expiration check wrong (`i-k`, not `i-k+1` comparisons done carelessly).
3. Using `<` instead of `<=` in back-pop and accidentally keeping stale duplicates too long.
4. Appending answer before window is full.

Mental trigger: when prompt says “for every window/subarray of size k, report best value,” think **monotonic deque** immediately.
