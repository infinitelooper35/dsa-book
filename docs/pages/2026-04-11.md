# Page 46 — Quickselect Without Panic: Partition Logic for Kth Largest

Yesterday we chose the right tool for “kth largest”: min-heap by default, Quickselect when asked for better average complexity.

Today we make Quickselect interview-safe.

The main failure mode isn’t coding speed. It’s index confusion. So anchor everything to one line:

`target = len(nums) - k`

That converts “kth largest” into the index you’d want in a sorted ascending array.

## Mental model in 20 seconds

Quickselect is partial quicksort:
- pick a pivot
- partition array into `< pivot` and `>= pivot` zones (or similar variant)
- check pivot’s final index `p`
  - if `p == target`, done
  - if `p < target`, search right side
  - if `p > target`, search left side

You only recurse/iterate into one side, which gives expected `O(n)` time.

## Interview-safe iterative template (Python)

```python
import random

def find_kth_largest(nums, k):
    target = len(nums) - k
    l, r = 0, len(nums) - 1

    while l <= r:
        pivot_idx = random.randint(l, r)
        p = partition(nums, l, r, pivot_idx)

        if p == target:
            return nums[p]
        elif p < target:
            l = p + 1
        else:
            r = p - 1


def partition(a, l, r, pivot_idx):
    pivot = a[pivot_idx]
    a[pivot_idx], a[r] = a[r], a[pivot_idx]  # move pivot to end

    store = l
    for i in range(l, r):
        if a[i] < pivot:
            a[store], a[i] = a[i], a[store]
            store += 1

    a[store], a[r] = a[r], a[store]
    return store
```

## What to narrate while coding

Use this script:
1. “I’ll randomize pivot to avoid consistently bad splits.”
2. “After partition, pivot is in final sorted position.”
3. “Then I discard half the search space based on pivot index vs target.”

This shows algorithmic control, not memorization.

## Edge cases interviewers probe

1. **Duplicates**: partition still works; target is index-based, not value-based.
2. **k = 1**: should return max element.
3. **k = n**: should return min element.
4. **Input mutation**: Quickselect mutates array — say this explicitly. If needed, copy first: `arr = nums[:]`.

## Heap vs Quickselect: final decision rule

- Need safest code under pressure: heap (`O(n log k)`).
- Need better average time and interviewer pushes optimization: Quickselect (expected `O(n)`).

If you can explain that tradeoff clearly, you’re already performing at a strong mid-to-senior interview level.
