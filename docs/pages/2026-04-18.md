# Page 53 — Jump Game VI: Max Score with DP + Monotonic Deque

Yesterday’s constrained-subsequence pattern gave you the right mental model: DP state plus “best value in a recent window.”
Today we apply it to a very interviewable variant: **Jump Game VI**.

Problem (LeetCode 1696 style):
You start at index `0`. From index `i`, you can jump to any `j` where `i < j <= i + k`. Score is the sum of visited values. Return the maximum score when you reach `n - 1`.

## Recurrence first (always)
Define `dp[i]` = max score to reach index `i`.

Transition:
`dp[i] = nums[i] + max(dp[j]) for j in [i-k, i-1]`

Brute-force transition scan is `O(k)` each step -> total `O(nk)`. Too slow when `n` and `k` are large.

## Optimization: sliding-window maximum on DP
You only need the max `dp` in the last `k` positions. Maintain a deque of candidate indices with **decreasing dp values**.

For each `i`:
1. Evict front while index is out of window (`< i-k`).
2. Front holds best previous state.
3. Compute `dp[i] = nums[i] + dp[dq[0]]`.
4. Pop back while `dp[back] <= dp[i]` (dominated states).
5. Push `i`.

Each index enters/exits deque once -> `O(n)` time, `O(n)` dp space (or `O(k)` with rolling tricks).

## Interview-grade Python

```python
from collections import deque

def max_result(nums, k):
    n = len(nums)
    dp = [0] * n
    dp[0] = nums[0]

    dq = deque([0])  # indices, dp decreasing

    for i in range(1, n):
        while dq and dq[0] < i - k:
            dq.popleft()

        dp[i] = nums[i] + dp[dq[0]]

        while dq and dp[dq[-1]] <= dp[i]:
            dq.pop()
        dq.append(i)

    return dp[-1]
```

## What to say while coding

“I’m using DP where each state depends on the best of the previous `k` states. I’ll maintain that best with a monotonic deque so I don’t rescan the window each time.”

That one sentence demonstrates both correctness and performance awareness.

## Mistakes that cost offers

1. Using a max-heap without lazy deletion handling (bug-prone under pressure).
2. Forgetting window eviction before reading best candidate.
3. Popping with `<` instead of `<=`, which keeps dominated duplicates and bloats deque.
4. Missing base case `dp[0] = nums[0]`.

Pattern memory: if recurrence says “max over last `k` DP states,” your default should be **DP + monotonic deque**.
