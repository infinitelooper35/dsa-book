Arrays

Article overview

  • Arrays are the first DSA concept to truly understand.
  • Most other data structures either build on arrays or fix their weaknesses.
  • Key focus: memory model, operation costs, and interview impact.

What is an array (or list)?

Foundational definition

  • An array stores multiple values in a specific order.
  • Everyday examples: TODO list, friends list, input sequence.
  • Conceptually simple, but performance behavior matters.

Creating an array blocks off a contiguous block of memory

Under-the-hood model

  • Arrays allocate one contiguous memory block for all elements.
  • Contiguity is the most important property to remember.
  • This enables fast access but makes middle edits expensive.

Indexing an array is O(1) time

Why constant-time access works

  • Address math: base + index * element_size.
  • No traversal is needed for arr[0], arr[5], or arr[n-1].
  • That is why indexing is constant-time.

Most programming languages provide a dynamic array implementation

Python list / Java ArrayList / C++ vector

  • Dynamic arrays still use contiguous memory.
  • When capacity runs out, runtime allocates a larger block and copies elements.
  • This resize cost is occasional, not constant.

Inserting at the end is O(1) amortized

Append behavior

  • Most appends are O(1): write into next free slot.
  • Resize moments cost O(n) due to copying.
  • Across many operations, average append cost is amortized O(1).

Inserting anywhere else is O(n)

Shift cost

  • Middle insertion requires shifting right-side elements.
  • Order and contiguity must be preserved.
  • Worst-case insertion away from end is linear.

Deleting from the end is O(1)

Cheap removal

  • Pop from end only decreases logical size.
  • No shifting is required.
  • This is constant-time.

Deleting from anywhere else is O(n)

Gap-filling shift

  • Middle deletion shifts right-side elements left.
  • The farther the gap, the more movement required.
  • Hence linear-time behavior.

Searching is O(n)… unless sorted

Linear vs binary search

  • Unsorted arrays require linear scan: O(n).
  • Sorted arrays enable binary search: O(log n).
  • Sortedness changes the search strategy completely.

Sorting is usually O(n log n)

General-purpose sorting

  • Typical practical sorts (QuickSort/MergeSort/HeapSort/Timsort) are O(n log n).
  • Quadratic sorts are mostly educational in modern practice.
  • Complexity awareness is critical in interviews.

Sometimes you can sort in O(n)

Constraint-driven optimization

  • If values are bounded (like small integer range), counting sort can reach O(n).
  • This is often the difference between pass and TLE.
  • Use constraints as design hints.

Know common array ops in your language

Python list operation costs

  • len: O(1), append: O(1) amortized, pop from end: O(1).
  • insert/remove/min/max usually involve O(n) behavior.
  • Operation-cost literacy improves solution quality before coding.

2D arrays (matrices)

Array of arrays

  • Used for grids, games, maps, image-like data.
  • Access remains O(1): grid[row][col].
  • Nested loops can quickly increase runtime.

Final takeaway

Why this matters

  • Master arrays first: indexing, shifting, and amortized behavior.
  • These mechanics appear repeatedly in interview problems.
  • Everything else in DSA builds on this mental model.
