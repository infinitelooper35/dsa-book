Section snapshot

What is a tree?

From article section

  • A tree is a hierarchical data structure that organizes data in a parent → child relationship.
  • One node at the top (the root) - Children branch downward - Real-world examples: file systems, org charts, family trees

Section snapshot

Core vocabulary

From article section

  • Node, root, edge, parent/child, leaf, subtree, ancestors/descendants, siblings, path.
  • Interviewers expect fluency with this language.

Section snapshot

Height vs depth

From article section

  • Depth: distance from root down to a node - Height: longest path from node down to a leaf When people say tree height, they usually mean root height.

Section snapshot

Trees are acyclic and connected

From article section

  • Trees have no cycles and exactly one path between root and any node.
  • These constraints separate trees from general graphs.

Section snapshot

Balanced vs unbalanced

From article section

  • Balanced trees preserve logarithmic behavior.
  • Unbalanced trees can degrade to linked-list-like O(n) behavior.

Section snapshot

Full vs complete binary trees

From article section

  • Full: each node has 0 or 2 children - Complete: levels filled left-to-right except possibly the last Heaps rely on complete binary tree structure.

Section snapshot

Defining a tree node

From article section

  • At the core, trees are nodes with pointers/references to children.
  • Binary trees use left/right pointers; N-ary trees use a children list.

Section snapshot

Depth-first search (DFS)

From article section

  • DFS explores depth before breadth.
  • Pre-order: node → left → right - In-order: left → node → right - Post-order: left → right → node Complexity: O(n) time, O(h) space.
IndexingO(1)
AppendO(1)*
Middle InsertO(n)
Binary SearchO(log n)

Breadth-first search (BFS)

From article section

  • BFS processes level by level using a queue.
  • Use for level-order traversal, shortest path in unweighted trees, min depth style questions.
  • Complexity: O(n) time, O(w) space.
IndexingO(1)
AppendO(1)*
Middle InsertO(n)
Binary SearchO(log n)

Common tree variations

From article section

  • Binary tree, N-ary tree, BST, Trie, Heap, Segment tree.
  • Most advanced problems are DFS/BFS plus constraints.

Know the common tree operations

From article section

  • Traversals: O(n) - Height/count nodes: O(n) - BST search/insert/delete: O(h) Balanced BSTs have h = O(log n), unbalanced can be O(n).
